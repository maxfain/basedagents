### MED-6: Webhook HMAC-SHA256 signing — PASS

**Files:** `packages/api/src/lib/webhooks.ts`, `packages/api/src/routes/register.ts`, `packages/api/src/routes/agents.ts`, `packages/api/src/routes/messages.ts`, `packages/api/src/routes/tasks.ts`, `packages/api/src/routes/verify.ts`, `packages/api/migrations/0022_add_webhook_secret.sql`

- `hmacSha256Hex()` uses Web Crypto API (`crypto.subtle`) for HMAC-SHA256 ✅
- `fireWebhook()` accepts optional `webhookSecret` parameter ✅
- Header format: `X-BasedAgents-Signature: sha256=<64-char-hex>` ✅
- Header omitted when secret is null/undefined (backward compatible) ✅
- `webhook_secret` generated on registration when `webhook_url` is provided ✅
- `webhook_secret` generated/rotated on profile update when `webhook_url` is set ✅
- `webhook_secret` cleared when `webhook_url` is cleared ✅
- All `fireWebhook` callers across messages, tasks, verify, register pass the secret ✅
- Secret is base64-encoded 32 random bytes (44 chars) ✅
- **Migration:** `ALTER TABLE agents ADD COLUMN webhook_secret TEXT DEFAULT NULL;` — valid SQLite syntax ✅
- **Tests:** 7 new tests cover HMAC presence/absence, determinism, different-key divergence, and secret lifecycle on profile update

**Bug found and fixed:** The test database setup (`test-helpers.ts`) was missing the `webhook_secret` column, causing 81 test failures (500 errors on any route that queries `webhook_secret`). Added `ALTER TABLE agents ADD COLUMN webhook_secret TEXT;` to `EXTRA_ALTER_STATEMENTS`.

### LOW-10: CSP headers added — PASS

**Files:** `packages/web/public/_headers`

- `script-src 'self' 'unsafe-inline'` — allows inline scripts (needed for most SPAs) ✅
- `style-src 'self' 'unsafe-inline'` — allows inline styles ✅
- `connect-src 'self' https://api.basedagents.ai` — restricts API calls to own domain + API ✅
