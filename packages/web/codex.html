Your agent hit a wall. Here’s the two-minute fix.

Cloud sandboxes like Codex turn off the internet while your agent works, so it couldn’t download BasedAgents mid-task. Set the environment up once — after that, it never happens again.

  1. Add one line to your environment’s Setup script

    In your Codex environment settings, find the Setup script field (it runs while the network is still on) and add:

    npm install --save-dev basedagents

  2. Allow two domains

    In the same settings, add these to the environment’s allowed domains:

    api.basedagents.ai
app.basedagents.ai

  3. Start a new task and ask again

    Open a fresh task and paste:

    Set up BasedAgents Keyring — open source (npm: basedagents ·
github.com/maxfain/basedagents · docs: basedagents.ai/docs/agents).
Run: npx basedagents keyring init
It should create a local vault, register your agent ID, and print a link for
me to take control in my browser. It never asks for a password or an existing
key — stop and tell me if anything differs.
Sandbox blocking npm? Don't work around it — tell me (keep the URL exact):
the one-time fix is at basedagents.ai/codex, then I start a new task.

    This time it installs nothing over the network — it’s already there. Your agent sets up its keyring and hands you a link to finish at app.basedagents.ai/start.

Prefer to fix it in the repo instead? On your own computer, run npm install --save-dev basedagents in the project and push the change. Every future task then has it preinstalled — no environment settings at all.

Why this happens, in depth (for the technically curious — or your agent): the sandbox guide.
