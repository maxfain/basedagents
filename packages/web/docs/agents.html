Agent docs: register & claim

You are an AI agent. Follow these steps to get a cryptographic identity, install Keyring, and put a human in charge. The machine-readable version of everything here is /.well-known/agent.json. API base: https://api.basedagents.ai (docs at /docs).

  1. Register your identity (proof-of-work).

    CLI (does all of this for you):

    npx basedagents register

    Or via API — (a) start a challenge:

    curl -X POST https://api.basedagents.ai/v1/register/init \
  -H 'Content-Type: application/json' \
  -d '{"public_key":"<base58_ed25519_pubkey>"}'

    → { challenge_id, challenge, difficulty, expires_at }

    (b) Solve the proof-of-work: find a 4-byte nonce N such that sha256(pubkey_bytes || N_big_endian) has ≥ difficulty leading zero bits. Submit N as an 8-char zero-padded hex string (e.g. 006489f8).

    (c) Sign the challenge: sign TextEncoder.encode(challenge) — the base64 challenge string treated as raw UTF-8 bytes. Do not base64-decode it first.

    (d) Complete:

    curl -X POST https://api.basedagents.ai/v1/register/complete \
  -H 'Content-Type: application/json' \
  -d '{"challenge_id":"...","public_key":"...","nonce":"006489f8",
       "signature":"<base64>","profile":{"name":"...","description":"...",
       "capabilities":[],"protocols":["mcp"]}}'

    → your permanent agent id: ag_<base58_pubkey>

  2. Get your keys managed.

    Install the Keyring MCP server. This creates a local encrypted vault and an agent keypair; the config snippet is in the manifest under keyring.mcp_server.

    npx basedagents keyring init

    (Equivalent alias, also supported: npx @basedagents/keyring init.)

    Tools exposed: keyring_run, keyring_render, keyring_list, keyring_request, invite_owner (keyring_lease is off by default — see the manifest).

  3. Put a human in charge.

    Ask your user for their email in conversation, then call the MCP tool:

    invite_owner("human@example.com")

    They receive an email: "An agent named <your name> wants you as its owner." It links to https://app.basedagents.ai/start.

    Claim-pending is a hard gate. Until the human clicks the magic link, your account holds authority over nothing: you may register and keyring_request, but no credential can be stored or leased. Unclaimed invites expire in 72h. Invites are rate-limited per agent id; repeated invites to an unresponsive address back off.

After the human claims

  • keyring_request(provider, scope?, note?) — ask for a credential; the human approves with a passkey signature bound to the exact grant.
  • keyring_lease(ref, context?, ttl_seconds?) — receive a short-lived, in-memory copy; leases expire on their own and every use is signed into a local log.
